# 架构与开发流程

## 1. 项目目标

本项目旨在实现一个轻量级的“餐厅推荐助手”原型，覆盖从前端输入、后端筛选到结果返回的完整流程。

## 2. 系统架构

### 2.1 组件划分

- Android 客户端：负责展示用户界面并发起推荐请求
- FastAPI 后端：接收请求并返回推荐结果
- 推荐服务层：根据筛选条件从数据源中挑选候选餐厅
- 数据层：使用 Yelp 商家数据样例，作为当前推荐逻辑的基础数据

### 2.2 请求流程

1. 用户在 Android 端输入偏好信息
2. 客户端向后端发送推荐请求
3. 后端服务根据城市、评分、距离、预算等条件筛选
4. 返回推荐结果列表和简短总结

## 3. 当前实现说明

### 后端接口

- POST /api/v1/recommendations
- 接收 JSON 请求体，返回推荐结果列表

### 推荐逻辑

当前版本主要基于以下条件进行筛选：

- 城市
- 最低评分
- 最大距离
- 预算约束
- 返回数量限制

## 4. 开发流程

### 本地开发步骤

1. 进入后端目录
2. 启动 FastAPI 服务
3. 使用 curl 或 Android 客户端调用接口
4. 根据返回结果调试推荐逻辑

### 推荐的迭代方向

- 增加更自然语言理解能力
- 将规则筛选替换为更智能的排序逻辑
- 接入数据库持久化与真实商家数据
- 增加多轮对话和上下文记忆

## 5. 适合这个项目的展示场景

- 课程项目展示
- 技术原型演示
- 智能助手方向的入门案例
- 后续扩展为更完整的推荐系统
